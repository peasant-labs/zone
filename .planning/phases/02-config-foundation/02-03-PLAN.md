   **renderAnnotatedTOML function:**
   Generate valid TOML output with inline comments showing source. Per user decision: annotated TOML with `# <source>` comment on each line.

   For scalar fields:
   ```
   version = 1 # repo: zone.toml

   [zone]
   harness = "claude-code" # repo: zone.toml
   base_image = "ubuntu:24.04" # global
   shell = "bash" # global (default)
   ```

   For merged list fields, per CONTEXT.md decision and research pitfall 4 (inline comments on array elements are invalid TOML), emit a comment block ABOVE the array:
   ```
   [packages]
   # apt: global provides ["git", "curl", "wget"]; repo adds ["build-essential"]
   apt = ["git", "curl", "wget", "build-essential"]
   ```

   Implementation approach: iterate through AnnotatedConfig fields. For each scalar AnnotatedField, emit `key = value # source`. For each list, group items by source, emit comment line, then array. Use `fmt.Fprintf` to write to a `strings.Builder`.

   For bool fields, emit `true`/`false`. For string fields, quote values. For int fields, emit raw number. For empty string fields, emit `""`. For empty lists, emit `[]`.

   Skip sections where all values are defaults and empty (don't clutter output with empty sections).

   **renderJSON function:**
   Build a nested map structure matching the TOML sections. Each leaf is `AnnotatedFieldJSON{Value: ..., Source: ...}`. For list fields, the value is the merged list and source describes the merge (e.g., "global+repo" or "repo" if replaced).

   Actually per user decision: "Structure: each field is `{ "value": ..., "source": "repo|global|default" }`". For lists where items come from different sources, use `"source": "merged"` or an array of source-tagged items. Simplest interpretation: for scalar fields use `{"value": X, "source": "repo"}`. For list fields use `{"value": [...], "source": "merged"}` with an optional `"items"` array showing per-item sources.

   Use `encoding/json.MarshalIndent(data, "", "  ")` for readable output.

   Print to stdout via `cmd.OutOrStdout()` (not `fmt.Println`) so tests can capture output.

   **RunE implementation:**
   ```go
   RunE: func(cmd *cobra.Command, args []string) error {
       // 1. Check for zone.toml
       repoPath := "zone.toml"
       if _, err := os.Stat(repoPath); os.IsNotExist(err) {
           cmd.PrintErrln("No zone.toml found. Run `zone init` to create one, or `zone config --global` to view global defaults.")
           return config.ErrNoConfig
       }

       // 2. Load and merge config, collecting errors along the way
       var allErrors config.ValidationErrors

       merged, _, err := config.LoadMerged(repoPath)
       if err != nil {
           // Check if it's an UnknownKeysError
           var uke *config.UnknownKeysError
           if errors.As(err, &uke) {
               allErrors = append(allErrors, config.ValidateUnknownKeys(uke.Keys, uke.File)...)
               // LoadMerged may still return a partial merged config via the UnknownKeysError
               // In practice, LoadRepo returns (*RepoConfig, *UnknownKeysError) where RepoConfig is populated
               // Need to handle this: call LoadMerged, if UnknownKeysError, still proceed with validation
               // This requires LoadRepo to return the config alongside the error
           } else {
               // Fatal parse error (not just unknown keys)
               cmd.PrintErrln(fmt.Sprintf("Error: %v", err))
               os.Exit(2)
               return err
           }
       }

       // 3. Run validation on merged config
       if merged != nil {
           validationErrs := config.Validate(merged)

           // Filter out dangerous_mount errors if --allow-dangerous-mount
           if allowDangerousMount {
               var filtered config.ValidationErrors
               for _, e := range validationErrs {
                   if e.Category != "dangerous_mount" {
                       filtered = append(filtered, e)
                   }
               }
               validationErrs = filtered
           }

           allErrors = append(allErrors, validationErrs...)
       }

       // 4. Report results
       if len(allErrors) == 0 {
           cmd.Println("zone.toml is valid.")
           return nil // exit 0
       }

       // Print all errors grouped by category
       cmd.PrintErrln(allErrors.Error())

       if allErrors.HasErrors() {
           os.Exit(2) // exit 2 for config errors per spec section 8
       }

       // Warnings only — exit 0 but print them
       return nil
   }
   ```

   **Important notes:**
   - Use `cmd.PrintErrln()` for error output (stderr)
   - Use `cmd.Println()` for success output (stdout)
   - Exit code 2 for config errors per spec section 8 (`DX-01: 2 = config error`)
   - `--allow-dangerous-mount` is a blanket flag per user decision (filters out all dangerous_mount category errors)
   - Unknown key errors from LoadMerged need special handling: LoadRepo returns the config alongside UnknownKeysError. Ensure LoadRepo in config.go returns `(*RepoConfig, error)` where RepoConfig is populated even when UnknownKeysError is returned. The executor should verify this behavior in config.go and adjust if needed (add a note to return the parsed config along with the unknown keys error).

   **Handle LoadRepo returning config + error:**
   The pattern in LoadRepo should be: parse succeeds, undecoded keys found -> return `(cfg, &UnknownKeysError{...})`. In validate.go, use `errors.As` to extract UnknownKeysError, then still use the parsed config for validation. If LoadMerged doesn't support this pattern directly, call LoadGlobal and LoadRepo separately:
   ```go
   global, globalErr := config.LoadGlobal()
   if globalErr != nil {
       // handle
   }
   repo, repoErr := config.LoadRepo(repoPath)
   var uke *config.UnknownKeysError
   if repoErr != nil && !errors.As(repoErr, &uke) {
       // Fatal error, not just unknown keys
       cmd.PrintErrln(fmt.Sprintf("Error: %v", repoErr))
       os.Exit(2)
   }
   if uke != nil {
       allErrors = append(allErrors, config.ValidateUnknownKeys(uke.Keys, uke.File)...)
   }
   merged, _ := config.Merge(global, repo)
   ```
   This approach gives full control. Use `config.Merge()` directly instead of `LoadMerged()`.

   Also check for global config unknown keys:
   ```go
   var gukeErr *config.UnknownKeysError
   if errors.As(globalErr, &gukeErr) {
       allErrors = append(allErrors, config.ValidateUnknownKeys(gukeErr.Keys, gukeErr.File)...)
   }
   ```
