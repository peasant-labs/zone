**Part A: Update main.go to translate ErrLockContention to exit code 5**

Replace the current main.go with:

```go
package main

import (
    "errors"
    "os"

    "github.com/peasant-labs/zone/cmd"
    "github.com/peasant-labs/zone/internal/cache"
)

var version = "dev"
var commit = "none"
var date = "unknown"

func main() {
    cmd.SetVersion(version, commit, date)
    if err := cmd.Execute(); err != nil {
        if errors.Is(err, cache.ErrLockContention) {
            os.Exit(5)
        }
        os.Exit(1)
    }
}
```

Key details:
- Import `"errors"` for `errors.Is`
- Import `"github.com/peasant-labs/zone/internal/cache"` for `cache.ErrLockContention`
- The `errors.Is` check works because `Lock.Acquire()` returns `fmt.Errorf("%w (PID %d)", ErrLockContention, pid)` — the `%w` verb wraps the sentinel, so `errors.Is` traverses the chain and finds it
- The check comes BEFORE the generic `os.Exit(1)` so it takes priority
- All non-lock-contention errors still fall through to `os.Exit(1)`

Add to `tests/cache_test.go`:

```go
// TestExitCodeLockContentionSentinel verifies that errors returned by Lock.Acquire()
// on contention are detectable via errors.Is(err, cache.ErrLockContention).
// This is the precondition for main.go's exit code 5 mapping.
// Full binary e2e test becomes possible once Phase 6 wires zone launch to call Lock.Acquire().
func TestExitCodeLockContentionSentinel(t *testing.T) {
    dir := t.TempDir()
    zoneDir := filepath.Join(dir, ".zone")
    require.NoError(t, os.MkdirAll(zoneDir, 0755))

    l1 := cache.NewLock(zoneDir)
    require.NoError(t, l1.Acquire())
    defer l1.Release()

    l2 := cache.NewLock(zoneDir)
    err := l2.Acquire()
    require.Error(t, err)

    // This is the exact check main.go uses to decide exit code 5
    assert.True(t, errors.Is(err, cache.ErrLockContention),
        "errors.Is(err, cache.ErrLockContention) must be true for main.go exit code 5 mapping")
}
```
