# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Project Scaffold.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Note:** Zone is a CLI-only tool (REQUIREMENTS.md Out-of-Scope: "GUI / desktop interface — CLI-only
> tool; target users are terminal developers"). Phase 1 has no web frontend, no browser UI, and no
> React/Tailwind stack. The design contract governs terminal output conventions only. shadcn and web
> design tokens are not applicable.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | REQUIREMENTS.md: CLI-only; no GUI |
| Preset | not applicable | No web UI in any phase |
| Component library | none | CLI output via Cobra + Lip Gloss (Phase 9 only) |
| Icon library | none | Terminal symbols only (ASCII/Unicode) |
| Font | Terminal monospace (host default) | CLI environment; no font control |

**Lip Gloss note:** Lip Gloss is listed in RESEARCH.md as a stub dependency in Phase 1 but is only
activated in Phase 9 (TUI Layer). Phase 1 produces no styled output — Cobra renders all help text
using its own built-in formatter.

---

## Spacing Scale

Terminal output uses character-column spacing, not pixel spacing. The 8-point scale is not applicable
to CLI help text. Column-based conventions apply instead.

| Token | Value | Usage |
|-------|-------|-------|
| indent-1 | 2 spaces | Cobra flag/command descriptions (Cobra default) |
| indent-2 | 4 spaces | Nested help sections |
| indent-3 | 6 spaces | Usage examples in `Example` field |
| column-gap | 2 spaces | Gap between flag name column and description column |

Exceptions: Cobra auto-formats column widths to terminal width; no manual padding required in Phase 1.

---

## Typography

Terminal output has no font size or weight control. Cobra manages all help text formatting.

| Role | Convention | Usage |
|------|------------|-------|
| Command name | UPPERCASE in synopsis: `ZONE` | Cobra `Use` field root |
| Flag names | `--flag-name` (double-dash, lowercase kebab) | All 4 global flags and per-command flags |
| Subcommand names | lowercase, no hyphens | `launch`, `init`, `stop`, etc. |
| Error messages | Plain prose, sentence case, no trailing period | `RunE` return value strings |
| Stub placeholder | `"not implemented"` | All 15 subcommand `RunE` returns — exact string |

Line length: no wrap enforcement in Phase 1. Cobra wraps to terminal width automatically.

---

## Color

Phase 1 produces no ANSI color output. All output is unstyled plain text. Lip Gloss is imported as
a stub dependency only; no `lipgloss.Style` calls are made.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | terminal default foreground | All help text, command listing |
| Secondary (30%) | terminal default foreground | Flag descriptions, aliases |
| Accent (10%) | none in Phase 1 | Reserved for Phase 9 TUI (Lip Gloss) |
| Destructive | none in Phase 1 | No destructive commands wired in Phase 1 |
| Error | terminal default (stderr) | "not implemented" error text to stderr via Cobra |

Accent reserved for: nothing in Phase 1. Lip Gloss color tokens are defined in Phase 9.

### Stub Error (Phase 1 Only)

All 15 subcommands return this exact error from `RunE`:

| Element | Copy |
|---------|------|
| Stub RunE error | `not implemented` |

This is the placeholder that all subsequent phases replace. It must be exactly `"not implemented"` —
no punctuation, no capitalization variance — so that Phase 8 integration tests can detect unimplemented
commands by scanning for this string.

### Empty State

Not applicable in Phase 1. No commands produce data output; all return the stub error.

### Error State (Cobra-level)

Cobra generates these automatically. Phase 1 does not override Cobra's default error formatting:

| Situation | Output |
|-----------|--------|
| Unknown command | `Error: unknown command "foo" for "zone"` (Cobra default) |
| Unknown flag | `Error: unknown flag: --foo` (Cobra default) |
| Missing required flag | `Error: required flag(s) "..." not set` (Cobra default) |
| RunE error | `Error: not implemented` (Cobra wraps RunE return) |

### Destructive Actions

None in Phase 1. The `destroy`, `stop`, `clean` commands exist as stubs only and return
`"not implemented"`. No confirmation dialogs or prompts are implemented here.

## Terminal Output Structure

This section supplements the standard template for CLI-specific contracts.

### `zone --help` Layout (Cobra default output)

```
Sandboxed Docker workspaces for LLM coding agents.

Zone generates and manages isolated Docker workspaces for LLM coding agents.
Run zone launch in any repo to get a sandboxed container with zero manual Docker configuration.

Usage:
  zone [command]

Available Commands:
  build       Force-rebuild the Docker image without launching
  clean       Remove the .zone/ cache directory
  config      Show effective merged configuration with source annotations
  destroy     Fully tear down container, image, network, and cache
  exec        Run a one-off command inside the running container
  init        Scaffold a zone.toml in the current directory
  join        Attach a new shell to a running container
  launch      Build (if needed) and attach to the container
  logs        View harness output
  ls          List all zone containers across all repos
  restart     Stop and relaunch the container
  shell       Open an interactive shell in the container
  status      Show container state, harness, uptime, and resources
  stop        Stop and remove the container and network
  validate    Check zone.toml validity without starting anything

Flags:
      --debug     Maximum verbosity including raw API responses
  -h, --help      help for zone
      --plain     Disable TUI and use plain text output
  -q, --quiet     Suppress all non-essential output
  -v, --verbose   Increase output verbosity

Use "zone [command] --help" for more information about a command.
```

### `zone launch` (stub) output

```
Error: not implemented
```

Exit code: 1 (Cobra default when RunE returns non-nil error).

Note: Phase 8 will map `fmt.Errorf("not implemented")` to the correct exit codes. Phase 1 stubs
intentionally use exit code 1 uniformly.
